# 2025 AI 前端工程师进阶全景大纲

> 完整无裁剪的《2025 AI 前端工程师进阶全景大纲》（共 22 章）。

## 🗺️ 全景大纲：从入门到架构师

### 第一阶段：AI 运行时基础 (The Runtime)

**目标**：懂得如何给 AI 装上“手”（Tools）和“眼”（RAG）。

| 章节   | 核心主题       | 关键技术点 (Key Tech)                      | 重要性          | 前端视角类比                                   |
| :----- | :------------- | :----------------------------------------- | :-------------- | :--------------------------------------------- |
| **01** | **架构总览**   | Agent = LLM + Memory + Tools + RAG         | ⭐️⭐️⭐️⭐️⭐️ | 理解 Vue/React 的生命周期与核心原理。          |
| **02** | **Tool 定义**  | Zod Schema, OpenAI Function Calling        | ⭐️⭐️⭐️⭐️    | 给组件定义 Props 类型 (TypeScript Interface)。 |
| **03** | **ReAct 循环** | while loop, Manual State Management        | ⭐️⭐️⭐️⭐️    | 手写一个简单的 Event Loop 或轮询机制。         |
| **04** | **环境交互**   | node:child_process, cwd 状态管理           | ⭐️⭐️⭐️       | 浏览器里的 window 对象与上下文管理。           |
| **05** | **MCP 协议**   | Client/Server 架构, Stdio 通信             | ⭐️⭐️⭐️⭐️⭐️ | 前后端分离，定义通用的 API 接口规范。          |
| **06** | **MCP 实战**   | 复用社区工具 (高德, Chrome, FS)            | ⭐️⭐️⭐️⭐️    | npm install 第三方组件库，不再造轮子。         |
| **07** | **RAG 原理**   | Embedding, Cosine Similarity, Vector Space | ⭐️⭐️⭐️⭐️⭐️ | 把关键词搜索升级为“语义哈希”匹配。             |
| **08** | **基础 ETL**   | WebLoader, RecursiveSplitter, Chunking     | ⭐️⭐️⭐️⭐️    | 简单的 DOM 解析与字符串 substring 处理。       |

### 第二阶段：夯实数据地基 (Advanced Data & Storage)

**目标**：解决“脏数据”和“失忆”问题。从 Demo 走向生产级数据处理。

| 章节   | 核心主题         | 关键技术点 (Key Tech)                      | 重要性          | 前端视角类比                                              |
| :----- | :--------------- | :----------------------------------------- | :-------------- | :-------------------------------------------------------- |
| **09** | **复杂文档加载** | PDF 解析, Office/Markdown 清洗, 表格还原   | ⭐️⭐️⭐️⭐️    | 像处理 dangerouslySetInnerHTML 里的脏 HTML 一样清洗数据。 |
| **10** | **向量数据库**   | Supabase (pgvector) / Pinecone, 持久化存储 | ⭐️⭐️⭐️⭐️⭐️ | 把 Redux 里的临时状态存进真正的数据库，刷新不丢失。       |
| **11** | **高级检索**     | Hybrid Search (混合检索), Rerank (重排序)  | ⭐️⭐️⭐️⭐️⭐️ | 搜索引擎的 SEO 优化，或者 CSS 的 z-index (决定谁排前面)。 |

| 章节   | 核心主题           | 关键技术点 (Key Tech)               | 重要性          | 前端视角类比                                           |
